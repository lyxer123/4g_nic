# 设备参数命令说明（蓝牙 BLE + 串口 UART）

本文描述如何通过 **低功耗蓝牙（GATT/JSON）** 与 **UART 交互式 CLI** 读取/设置与网页部分等效的参数。  
固件侧实现：`components/ble_settings/`（BLE）、`components/serial_cli/`（串口，基于 ESP-IDF **console** 组件 + **linenoise**）。

---

## 一、蓝牙 BLE

### 传输与 UUID

| 项 | UUID（16 位） | 方向 |
|----|----------------|------|
| 服务 | `0xFF50` | — |
| RX | `0xFF51` | 主机 → 设备（写 JSON） |
| TX | `0xFF52` | 设备 → 主机（通知，UTF-8 文本） |

广播名一般为 **`4G_NIC_CFG`**。协议细节见 [`ble_protocol.md`](ble_protocol.md) / [`ble_protocolcn.md`](ble_protocolcn.md)。

### JSON 命令（写入 RX）

每条消息为 **一个 UTF-8 JSON 对象**，且包含字符串字段 `"cmd"`。

| `cmd` | 附加字段 | 设备通过 TX 通知返回（示例） |
|-------|----------|------------------------------|
| `ping` | — | `{"ok":true,"cmd":"ping","device":"4g_nic"}` |
| `get_mode` | — | `{"ok":true,"work_mode_id":1}` |
| `set_mode` | `work_mode_id`（数字） | `{"ok":true,"cmd":"set_mode","work_mode_id":2}` 或带 `error` / `esp` |
| `version` | — | `{"ok":true,"version":"…"}` |

`set_mode` 与网页 POST `work_mode_id` 使用相同校验与延期应用逻辑（`web_service_apply_work_mode_id`）。

### 工具

- **手机（Android）**：推荐 **nRF Connect**；备选 **BLE Scanner**、**Bluetooth LE Scanner** 等（选下载量多、近期有更新的即可）。
- **PC**：仓库内 `PCSoftware`（Python + bleak）、**nRF Connect for Desktop**、**Bluetooth LE Explorer**（微软商店）。

## 二、串口 UART CLI

### 组件说明

使用 **ESP-IDF `console` 组件** 的 **`esp_console_init` + `esp_console_cmd_register` + `esp_console_run`**。  
**未使用** `esp_console_new_repl_uart`（linenoise REPL）：在已与 `esp_log` 共用 **UART0** 的工程里，REPL 会接管 stdio，易与日志形成 **重入**，曾导致 Guru/MMU 异常。

实现：独立任务用 **`uart_read_bytes`** 读一行再 **`esp_console_run`**，应答用 **`uart_write_bytes`**，避免经 libc `printf` 与日志抢同一输出链。

在 `menuconfig` 中由 **`CONFIG_ESP_CONSOLE_*`** 决定 UART 编号与波特率。  
说明：**日志**往往在未调用 `uart_driver_install()` 时就能从 UART0 打出（ROM/早期路径），此时 **`uart_is_driver_installed(0)` 仍为 false**，但串口在“发送日志”意义上是工作的；**读键盘输入**必须安装带 **RX 队列** 的驱动，因此 `serial_cli` 会在需要时自行 **`uart_driver_install`**（若已安装则复用）。

- 常见：**UART0**，波特率 **115200**（与 `idf.py monitor` 一致）。
- 若工程改为 **USB Serial JTAG** 作为控制台，则需相应调整（当前实现绑定 `CONFIG_ESP_CONSOLE_UART_NUM`）。

### 使用方式

1. 用串口终端连接设备（如 `idf.py monitor`、PuTTY、MobaXterm），波特率与工程配置一致（默认 **115200**）。
2. 启动后出现提示符 **`4g_nic> `**（日志与 CLI 共用同一端口时，输出可能交错，属正常现象）。
3. 输入命令后回车；输入 **`help`** 查看已注册命令。

### 命令列表

| 命令 | 说明 | 示例 |
|------|------|------|
| `help` | 列出命令及简要说明（console 内置） | `help` |
| `ping` | 存活检测 | `ping` |
| `mode_get` | 读取当前 `work_mode_id`（NVS，与网页 GET 一致） | `mode_get` |
| `mode_set` | 设置 `work_mode_id` 并延期应用（与网页 POST 一致） | `mode_set 3` |
| `version` | 打印应用版本（`esp_app_desc`） | `version` |

成功行一般以 `ok` 开头；失败以 `ERR` 开头并附原因或 `esp_err` 名。

### 与 BLE 的对应关系

| 能力 | BLE（JSON `cmd`） | 串口 |
|------|-------------------|------|
| 存活检测 | `ping` | `ping` |
| 读工作模式 | `get_mode` | `mode_get` |
| 写工作模式 | `set_mode` + `work_mode_id` | `mode_set <id>` |
| 版本 | `version` | `version` |

后续若在网页/BLE 增加更多字段，可在 `ble_settings.c` 增加 JSON `cmd`，在 `serial_cli.c` 增加同名语义的子命令，并同步更新本文件。

## 三、安全提示

当前 BLE 与串口 CLI **均无应用层鉴权**（串口具备物理接触即可）。生产环境请结合现场需求增加鉴权、配对策略或禁用未使用入口。
