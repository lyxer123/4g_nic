# 时区设置修复

## 问题描述

`AT+MODEMTIME` 返回的时间与计算机本地时间相差8小时。

示例：
```
+MODEMTIME:26/04/16,17:15:51+32
```

## 问题分析

1. **4G模组返回的时间**：`17:15:51+32`
   - `+32` 表示 UTC+8（32 × 15分钟 = 480分钟 = 8小时）
   - 这是**网络时间**，已经包含了时区信息

2. **ESP32-S3系统时间**：
   - 固件之前**没有设置时区**
   - 系统默认使用 **UTC时间**
   - 所以 `AT+TIME` 返回的是UTC时间，比北京时间慢8小时

3. **时间差异**：
   - 模组时间：17:15:51 (UTC+8)
   - 系统时间：09:15:51 (UTC)
   - 差异：8小时

## 解决方案

### 修改文件：`main/app_main.c`

**添加时区设置**：

```c
#include <stdlib.h>
#include <time.h>

void app_main(void)
{
    /* Set timezone to China Standard Time (UTC+8) */
    setenv("TZ", "CST-8", 1);
    tzset();
    
    // ... 其他初始化代码
}
```

### 时区字符串说明

- `CST-8` 表示 **China Standard Time, UTC+8**
- 格式：`标准时区名称-偏移小时数`
- 注意：**偏移量是负数**，因为POSIX时区格式中，东区为负

### 其他常用时区

```c
// 北京时间 (UTC+8)
setenv("TZ", "CST-8", 1);

// 日本时间 (UTC+9)
setenv("TZ", "JST-9", 1);

// 美国东部时间 (UTC-5)
setenv("TZ", "EST5EDT,M3.2.0,M11.1.0", 1);

// UTC时间
setenv("TZ", "UTC0", 1);
```

## 相关AT指令

| 指令 | 说明 | 时区 |
|------|------|------|
| `AT+TIME` | 系统时间 | 使用TZ环境变量 |
| `AT+MODEMTIME` | 4G模组网络时间 | 模组自动获取 |
| `AT+TIME=YYYY-MM-DD HH:MM:SS` | 设置系统时间 | 使用TZ环境变量 |

## 注意事项

1. **时区设置必须在 `tzset()` 之前**
2. **时区字符串格式遵循POSIX标准**
3. **如果需要支持夏令时，需要使用更复杂的时区字符串**
4. **NVS中保存的时间不受时区影响（存储的是UTC时间戳）**
